# Repositories

The Arkitekt client stack is devided into multiple repositories. This is done to keep the codebase clean and to allow for a more modular approach to the client stack.
However this can make it difficult to get an overview of the entire stack. This page will give you an overview of the different repositories and how they are connected.
linked to funcionality provided by the framework. If you are looking for a specific feature, you can find it here, and if you encounter
bugs this might give you an idea of where to look.
